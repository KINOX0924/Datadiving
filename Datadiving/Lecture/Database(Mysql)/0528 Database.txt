5월 28일 강의

ORM (Object Relation Mapping)
- 객체 지향 언어와 관계형 DB 모델을 연결하는 프로그램 기법
- Django(파이썬 웹 프레임워크)
- Jpa(Java)
- 파이썬에서는 SQLAlchemy 가 ORM 을 지원하고 있음
- 전체 테이블이 10 개 미만일때는 ORM 방식이 편함
- 테이블 세개 이상 조인을 할 시 속도 문제 및 쿼리(테이블간의 구조) 가 복잡해질 수록 ORM 만으로는 처리가 되지 않음

SPA (Single Page Application)
- 인스타그램 , 페이스북 등의 개발기법
- 웹페이지를 하나만 만들어서 페이지 체인지를 하는 방식으로 화면이동이 훨씬 매끄럽고, 무한 스크롤을 지원함
- 인스타그램 개발하다가 만들게 된 react 라이브러리가 대표적(그 외 vuejs(한국 금융권) , angular , polymer(유튜브))
- SPA 방식 웹 개발 시 ORM 방식을 많이 사용함


SQLite
- 본래 의미의 DB 는 아니고 사실은 파이썬의 file 과 같은 것이나
쿼리형식으로 읽고 쓰기가 가능함, 하지만 네트워크가 안되기에 휴대폰의 전화번호부 같은 형태로만 제한적으로 사용됨
